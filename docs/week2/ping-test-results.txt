# Test Plan – Hybrid Cloud IPsec VPN

## 1. Test Objectives
- IPsec tunnel ko verify karna on-premise aur cloud ke beech
- Dono sides ko ek dusre ko ping karna
- Wireshark mein ESP encrypted packets capture karna

## 2. Test Environment

| Device | IP Address | Role |
|--------|------------|------|
| R1 | 172.16.1.1 | On-premise router |
| PC1 | 172.16.1.10 | On-premise server |
| Cloud Gateway | 10.0.0.1 | Cloud router |
| Cloud Server | 10.0.0.10 | Cloud VM |

## 3. Test Cases

### TC-01: On-premise to Cloud Ping
**Steps:**
1. IPsec tunnel up hai confirm karo
2. R1 console mein `ping 10.0.0.1` likho
3. Reply check karo

**Expected Result:** `!!!!!` ya `Reply from 10.0.0.1`

### TC-02: Cloud to On-premise Ping
**Steps:**
1. Cloud gateway console mein `ping 172.16.1.1` likho
2. Reply check karo

**Expected Result:** Ping successful

### TC-03: Wireshark ESP Packet Capture
**Steps:**
1. Wireshark mein vEthernet interface select karo
2. Filter mein `esp` likho
3. Start capture karo
4. Ping bhejo dono taraf
5. ESP packets capture honge

**Expected Result:** Green/blue ESP packets dikhne chahiye

### TC-04: Show Crypto IPsec SA
**Steps:**
1. R1 console mein `show crypto ipsec sa` likho
2. Output check karo

**Expected Result:** SPI, encryption algorithm, tunnel details dikhni chahiye

## 4. Pass/Fail Criteria
- **Pass:** Ping reply aaye, ESP packets capture ho, SA details dikhe
- **Fail:** Timeout ho, ESP packets na dikhe, SA empty ho

## 5. Tools Required
- GNS3 (R1 router, PC1)
- Wireshark
- VPCS
